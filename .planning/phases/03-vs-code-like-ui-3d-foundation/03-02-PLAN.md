  <name>Task 2: Cliente api.ts + fluxo job/elicit</name>
  <files>services/web/src/lib/api.ts, services/web/src/hooks/useJobFlow.ts</files>

  <action>
Criar **`api.ts`** com:

- `postIntentElicit(prompt: string, attempt?: number)` → `POST /api/v1/intent/elicit`
- `postJob(body: IntentEnvelopeJson)` → `POST /api/v1/jobs` (tipar `body` como objeto JSON serializável compatível com exemplo README fase 2)
- `getJob(id: string)` → `GET /api/v1/jobs/{id}`

Base URL: `import.meta.env.VITE_API_BASE_URL`.

Criar **`useJobFlow.ts`** hook:

- Estados: `phase` (`idle`|`eliciting`|`creating`|`polling`|`done`|`error`), `lastResponse`, `jobId`, `job`, `error`.
- Fluxo opcional de demonstração **A**: elicit → construir envelope mínimo válido a partir de `ElicitSuccess` (parse JSON) → criar job → **poll** `GET` a cada 2s até terminal (`success|failed`) com cancel cleanup.
- Permitir **modo simplificado B** (env sem elicit) se elicit falhar falta API key — documentar código com comentário + ramo que envia apenas JSON canónico de teste já em fixtures TypeScript literal alinhado a `CANON_JSON` (pode copiar valores de `services/api/tests/test_intent_schema.py` excerpt).

Painel inferior mostra texto legível do estado + último erro JSON se 422.

**Polling:** `setInterval` ou `AbortController` — limpar sempre no `useEffect` cleanup.

  </action>
